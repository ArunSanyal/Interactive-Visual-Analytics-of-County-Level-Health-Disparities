Description

This project is an interactive coordinated dashboard for exploring county-level health disparities using the 2025 County Health Rankings dataset. It is built in Plotly Dash and designed to complement the standard choropleth map with additional views that address well-known choropleth trade-offs such as area-size bias, classification sensitivity, and the limitation of showing only one variable at a time. The dashboard focuses on seven health and social determinant variables including premature death rate, physically unhealthy days, mentally unhealthy days, fair or poor health, uninsured rate, unemployment, and children in poverty. It supports four classification methods (quantile, equal interval, natural breaks, and standard deviation), a scatter plot and parallel coordinates view with coordinated selection, and an equal-area H3 hex view that mitigates the visual dominance of large counties.

How to Run

1. Download the required data files (see Data Sources below) and place them in the data/ folder.
2. pip install -r requirements.txt
3. python scripts/preprocess.py
4. python app.py
5. Open http://127.0.0.1:8050 in a browser.

Data Sources

The data files are too large for GitHub and must be downloaded manually before running the preprocessing step.

1. County Health Rankings (2025)
   Source: University of Wisconsin Population Health Institute
   File needed: 2025 County Health Rankings Data - v4.xlsx
   Download from:
   https://www.countyhealthrankings.org/health-data/methodology-and-sources/data-documentation
   Under NATIONAL DATA, click "2025 County Health Release National Data" to download the Excel file.
   Place in: data/
2. U.S. Census TIGER/Line County Boundaries (2025)
   Source: U.S. Census Bureau
   Files needed: tl_2025_us_county.shp, .shx, .dbf, .prj, .cpg
   Download from:
   https://www.census.gov/cgi-bin/geo/shapefiles/index.php?year=2025&layergroup=Counties+(and+equivalent)
   Click Download.
   Unzip and place all files in: data/

After downloading, your data/ folder should contain:

```
data/
  2025 County Health Rankings Data - v4.xlsx
  tl_2025_us_county.shp
  tl_2025_us_county.shx
  tl_2025_us_county.dbf
  tl_2025_us_county.prj
  tl_2025_us_county.cpg
```

Dashboard Features

Choropleth map with classification dropdown — users can switch between quantile, equal interval, natural breaks (Jenks), and standard deviation groupings to see how classification choices change the visual pattern on the same data. County-level tooltips show exact values on hover.

Distribution histogram — updates with the selected metric, with vertical dashed lines marking class breaks and bars coloured by class so the user can see what range each map colour represents.

Scatter plot — bivariate view with user-selectable X and Y axes, supports lasso and box selection.

Parallel coordinates — displays all seven variables side by side with selected counties highlighted in blue.

Coordinated selection — selection made in any view propagates to all others through a shared selection store, with a reset button to clear the selection.

H3 hex view toggle — replaces county polygons with equal-area hexagons computed using H3 at resolution 4, so that large sparse counties and small dense counties get equal visual weight, directly mitigating area-size bias.

Notes

The preprocessing script joins the CHR Excel data to Census TIGER/Line shapefiles using the 5-digit FIPS code. State-level aggregate rows (FIPS ending in 000) are excluded.

Counties with missing or suppressed data appear transparent on the map, as "No data" in the details panel, and are excluded from plots that require numeric values.

If CHR column names change across data releases, update the column positions and metric definitions in src/config.py and re-run preprocess.py.

The project was tested in Google Chrome.
